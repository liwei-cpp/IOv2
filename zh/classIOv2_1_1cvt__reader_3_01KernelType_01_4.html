详细描述

template<io_converter KernelType>
requires (std::is_base_of_v<root_cvt<typename KernelType::device_type, true>, KernelType> && !is_mem_device<typename KernelType::device_type>)
class IOv2::cvt_reader< KernelType >

cvt_reader 针对具有内部缓冲区的 root_cvt（HasInBuffer = true，非 mem_device）的偏特化。 直接复用 root_cvt 内部缓冲区，避免额外的数据拷贝。

模板参数
KernelType须派生自 root_cvt<DeviceType, true> 且 DeviceType 非 mem_device。

cvt_reader 针对 mem_device 的偏特化（适用于 rb_root_cvt 和 no_rb_root_cvt 两种内核）。 mem_device 直接暴露内部缓冲区指针，因此所有操作直接转发给设备，无需额外的缓冲层。

模板参数
KernelType须派生自 root_cvt<DeviceType, true|false> 且 DeviceType 须为 mem_device。

template<io_converter KernelType>
template<bool Saturate = false>
auto IOv2::cvt_reader< KernelType >::get_buf ( size_t  to_max)
inline

从 root_cvt 的内部缓冲区获取数据，缓冲区不足时自动从设备读取补充。

模板参数
Saturate控制数据不足时的行为：
  • false（默认）：返回当前可用数据（可能少于请求量），返回类型为 std::pair<const char_type*, size_t>。
  • true：要求恰好 to_max 个字符，不足则抛异常，返回类型为 const char_type*。
参数
to_max要读取的字符数，不得超过 s_buffer_length。
返回
Saturate=false 时返回 (指针, 实际数量) 对；Saturate=true 时返回恰好 to_max 个字符的起始指针。
异常
cvt_error
  • to_max 为 0 时。
  • to_max 超过 s_buffer_length 时。
  • io_status 不为 input 且无法切换时。
  • Saturate=true 时在读满前遇到流末尾。
